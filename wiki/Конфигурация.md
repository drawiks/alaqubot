# ⚙️ Конфигурация плагинов

## Формат YAML

Конфигурация плагинов хранится в YAML файлах:

```
src/plugins/<имя_плагина>/config.yaml
```

## Структура конфига

```yaml
enabled: true              # Включен/выключен плагин

commands:                 # Настройки команд
  команда1:
    cooldown: 10         # Кулдаун в секундах
    enabled: true        # Включена ли команда
    requires_permission: false  # Требуются ли права

settings:                # Кастомные настройки
  option1: value
  option2: value
```

## Параметры команд

| Параметр | Тип | Описание |
|----------|-----|----------|
| `cooldown` | int | Кулдаун в секундах |
| `enabled` | bool | Включена ли команда |
| `requires_permission` | bool | Только для модеров |

## Примеры

### Простой конфиг

```yaml
enabled: true

commands:
  привет:
    cooldown: 0
    enabled: true
```

### Конфиг с правами

```yaml
enabled: true

commands:
  модер_команда:
    cooldown: 0
    enabled: true
    requires_permission: true
```

### Конфиг с кастомными настройками

```yaml
enabled: true

commands:
  зона:
    cooldown: 10
    enabled: true

settings:
  words:
    - "тихоня"
    - "лох"
    - "мент"
  custom_greeting: "Привет, {user}!"
```

## Отключение плагина

Чтобы отключить плагин, измени `enabled`:

```yaml
enabled: false  # Плагин не загрузится
```

## Отключение команды

Чтобы отключить команду:

```yaml
commands:
  команда:
    enabled: false  # Команда не будет зарегистрирована
```
