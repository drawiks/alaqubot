# 📦 Распространение плагинов

## Варианты распространения

### Вариант 1: GitHub репозиторий

Создай отдельный репозиторий для плагинов.

#### Структура репозитория

```
alaqubot-plugins/
├── README.md
├── LICENSE
├── plugins/
│   ├── my-plugin/          # Плагин 1
│   │   ├── __init__.py
│   │   ├── plugin.py
│   │   └── config.yaml
│   └── another-plugin/     # Плагин 2
│       ├── __init__.py
│       ├── plugin.py
│       └── config.yaml
└── INSTALL.md
```

#### INSTALL.md

```markdown
# Установка плагинов

## Ручная установка

1. Скопируй папку плагина в `src/plugins/`
2. Перезапусти бота

## Клонирование репозитория

git clone https://github.com/твой-ник/alaqubot-plugins.git

# Скопируй нужные плагины
cp -r plugins/my-plugin/* ../alaqubot/src/plugins/
```

### Вариант 2: GitHub Gist

Для маленьких плагинов используй Gist.

1. Создай Gist с файлами плагина
2. Поделись ссылкой
3. Другие пользователи скачивают и устанавливают вручную

### Вариант 3: Pull Request

Добавь свой плагин в основной репозиторий.

1. Форкни репозиторий
2. Добавь плагин в `src/plugins/`
3. Создай Pull Request

---

## Что включить в плагин

### Обязательно

- `__init__.py` - точка входа
- `plugin.py` - код плагина
- `config.yaml` - конфигурация

### Рекомендуется

- `README.md` - описание плагина
- `LICENSE` - лицензия (по умолчанию MIT)

---

## Лучшие практики

### 1. Версионирование

```python
class MyPlugin(Plugin):
    name = "my-plugin"
    version = "1.0.0"  # Семантическое версионирование
```

### 2. Обработка ошибок

```python
async def my_command(self, cmd):
    try:
        # Код команды
        await cmd.reply("Результат")
    except Exception as e:
        await cmd.reply("Произошла ошибка!")
        # Логирование
```

### 3. Документация

- Опиши все команды
- Укажи требования
- Приведи примеры использования

### 4. Тестирование

Проверь плагин перед публикацией:
- Команды работают
- Права проверяются
- Конфиг загружается
